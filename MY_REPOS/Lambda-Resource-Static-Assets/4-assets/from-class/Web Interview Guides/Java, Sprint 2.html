Interview PromptExpectations
3
Can you explain your data model, data schema to me?Student's data model includes all requirements as follows:
- All tables have a primary key.
- Model avoids duplication and makes use of annotations to express one to many relationships.
- Data types used are appropriate for each case.
- Required fields are marked as NOT NULL.
- Cascading turned on for One to Many relationships
4
Can you explain how you connected your API to a database?Student configures Models to represent the database, repositories to access the database to the Java Application and services to manipulate the data.
5
Can you explain the flow of data from client through the backend system and back to the client?Student explains the flow of information from the client through the system back to the client correctly but may no explicitly use the correct terms. They may explain the process without saying request, response, Spring Data, Tomcat, etc.
6
How did you handle querying in your application: custom querying vs JPA Querying?- Student uses a combination of Spring Data JPA and custom queries to access and manipulate multiple tables in their API AND,
- Student can explain where each type of query is used and the advantages of using that type.
- Note if a student does NOT use custom queries and can explain why, that is an acceptable answer.
