Can you explain how you connected your API to a database?Student configures API to connect to the correct database using Knex. Specifically, student has Knex installed, the knexfile.js present and have the configuration inside that file correct.

Can you show me the endpoints that preform crud operations and explain how they are connected to the database?Student's API has endpoints that perform CRUD operations and persist data to the database.
5
Can you explain your data model to me? How did you make sure to follow client requirements?
Student's data model includes all requirements as follows:
- All tables have a primary key.
- Model avoids duplication and makes use of foreign keys to express one to many relationships.
- Data types used are appropriate for each case.
- Required fields are marked as NOT NULL.
- Foreign Keys have onDelete and onUpdate cascading defined.
6
Can you show me your data migrations?Student's migrations reflect the model design as outlined in project ReadMe and include the down function to undo the changes.
7
How did you work with multiple tables in your code?Student's data access code uses JOINS to retrieve data where necessary, AND code considers the third table when saving data to reflect many to many relationships between tables
